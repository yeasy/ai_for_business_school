## 本章小结

本章沿着“用什么造—去哪儿买—怎么谈—放哪儿跑—怎么验收”的决策链，把智能体的供给侧生态过了一遍。核心结论如下：

- **开源框架分两层用**。代码框架中，LangChain/LangGraph 企业采用最广，核心卖点是持久化状态管理与人在回路；CrewAI 以角色协作范式上手最快；微软 Agent Framework 整合了 AutoGen 与 Semantic Kernel 两条脉络；Dify 等低代码平台让业务团队也能动手。常见打法是低代码验证价值、代码框架承接生产。框架迭代极快，任何选型结论的保质期都以半年计，决策时以官方文档现状为准。
- **商业平台跟着系统栈走**。国际市场四大阵营各守生态：CRM 原生的 Agentforce、依托 M365 的 Copilot Studio、统一入口的 Gemini Enterprise、中立运行时路线的 AWS Bedrock AgentCore；国内则是“云厂商＋模型厂”一体的百炼、扣子、千帆、元器等。平台能力在收敛，迁移成本是实打实的——先问数据和员工在哪里，再看平台演示。同时留意 SaaS 软件正在原地“长出”智能体：采购前先盘点现有供应商的路线图，但也要清醒，内嵌智能体给的是全行业的及格线，做不了跨系统的流程重构。
- **选型谈判守住三底线**。研究显示（MIT 2025 访谈样本口径），专业厂商主导的项目成功率约 67%、纯自建约 33%——对多数企业，合理姿态是“能买不建，买了会验，验的标准自己定”。工具是供应商评估五问（行业案例、数据归属、可迁移性、报价结构、退出条款）与 POC 三要点（真实数据、量化 KPI、时间盒）；红线是数据、流程、评估权必须留在自己手里。战略层面的自建与采购之辨，用 10.4 的 2×2 框架。
- **数据安全靠分级匹配**。“数据放出去安不安全”要拆成两步回答：数据分四级（公开、内部、敏感、核心），部署分三档（公有云 API、专有云、私有化开源模型），逐场景匹配而非一刀切；中国企业再叠加数据出境、行业监管与服务备案三层合规。2026 年，开源模型私有化已是可正经评估的选项，但要算清能力代差与长期运维这笔账。
- **质检是一条产线而非一次动作**。概率性输出让传统软件测试失效：上线前靠源自真实失败的黄金评测集与校准过的 LLM 评审员，上线后靠四类指标监控、漂移检测与全链路追踪，失败案例持续回流评测集，构成 AgentOps 闭环。评测集既是质检工具，也是验收谈判的筹码，更是行业经验沉淀为数字资产的具体形态。

一句话收束本章：平台可以租，模型可以换，供应商可以替——唯有数据、流程与评估权，从第一天起就必须留在自己手里。
