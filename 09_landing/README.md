# 第九章 落地方法论：从试点到规模化

第七章算清了"值不值"这笔账，第八章看过了"别人做成了什么"。本章回答落地篇最后、也最难的一个问题：怎么把 AI 从一次亮眼的试点，做成可规模化的经营能力——以及，做不成的时候怎么体面地收手。

本章也是全书理论主线的落点。[第 1.2 节](../01_essence/1.2_llm_base.md)提出大模型是新的通用目的技术，并预告其早期回报会被互补性投资吸收；本章用生产率 J 曲线把这条主线收拢：9.1 节直面"约 95% 的试点未形成可衡量利润影响"这组数字，给出口径、局限与经济学解释——败因几乎不在技术，而在互补性投资没有完成；9.2 节展开其中最大的一块互补投资——数据就绪，给出四维自查与补课顺序；9.3 节讨论第二块——工作流是"优化"还是"重构"，为什么打补丁式的 AI 助手换不来组织级回报；9.4 节给出可立即执行的起步五步法，附管理者亲手跑通第一个任务的路径；9.5 节回答"敢不敢放权"——像带新人一样带 AI 的四条可信可控原则与信任爬坡节奏；9.6 节补上多数方法论缺失的最后一环——止损的纪律：预设退出条件、识别叫停信号、把学费沉淀成资产。

分工边界需要事先说明。其一，"治理"分三层：单个智能体的技术质检（评测集、监控）在[第 6.5 节](../06_ecosystem/6.5_evaluation.md)；本章 9.5 节只讲项目级的信任爬坡；企业级制度（责任、审计、合规）在[第 12.3 节](../12_governance/12.3_governance_system.md)，9.5 节末尾会写明四原则如何升格为制度。其二，四类成本的算账框架保留在[第 7.3 节](../07_value/7.3_cost_benefit.md)，本章 9.2 节只展开数据这一块的操作方法。其三，自建还是采购的战略决策权在[第 10.4 节](../10_strategy/10.4_decision_matrix.md)，本章 9.4 节仅主张"起步阶段先用现成工具"。
