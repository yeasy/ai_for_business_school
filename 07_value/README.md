# 第七章 算清这笔账：价值与成本

前六章回答了智能体“是什么、为什么是现在、靠什么技术、怎么选平台”。从本章起，全书进入落地篇，而落地的第一个问题就是钱：AI 到底在哪里创造价值，需要付出什么代价，这笔账该怎么算才不失真。

本章按“先看清价值往哪里走，再逐项算账”的顺序展开。7.1 从交易成本理论出发解释“价值迁移”：执行性工作的成本趋近于零，且对所有竞争者同步下降，真正升值的是专有数据、判断力与渠道信任——这为全书讨论企业边界与自建、采购决策提供学理基础。7.2 把 AI 的价值拆成三张面孔——降本、增长、重估，并用写进财报口径的实证说明“AI 不只省钱，更能多赚”。7.3 给出算账的标尺：四类成本（建设、数据、运营、风险）对三类收益（降本、提速、增收筑壁垒）；其中数据这一块最大也最费工，怎么评估与建设数据就绪度不在这里展开，放到了 [9.2](../09_landing/9.2_data_readiness.md)，本章只安顿它在账本上的位置。7.4 回答全书被问得最多的问题——到底要花多少钱，按试点、扩展、规模化三档给出量级区间与成本构成的方向性判断，并提示按 token 计费的用量正在成为需要配额管理的新成本科目。7.5 提供“高频、结构化、可审计”三把经济尺子甄别真机会与伪需求——这是一把“经济判据”，与 1.3 的商业层现象、2.4 的任务判据、4.3 的技术成因合成同一条能力边界的四个层面，落到 7.5 时回引 2.4 而不再重述。

需要先交代一句的是，本章只管价值与成本这一侧：供应商怎么挑、合同怎么谈在 [6.3](../06_ecosystem/6.3_sourcing.md)，某个场景到底自建还是采购的战略决断在 [10.4](../10_strategy/10.4_decision_matrix.md)，投入节奏与向董事会讲账在 [10.5](../10_strategy/10.5_pacing_reporting.md)——本章为这几处备好算账的依据，不代它们拍板。
