# 第二章 智能体：能交付的数字员工

第一章把这波浪潮的商业本质讲清楚了：AI 完成了从"会看""会写"到"会干活"的三级跳，而承载"会干活"这一跳的载体，就是智能体（AI Agent）。本章的任务，是把"智能体"这个被过度使用的词彻底讲明白——它到底是什么、像什么、能用在哪里。

全章围绕四个递进的问题展开。2.1 节回答"是什么"：为什么说能回答的只是助手、能交付的才是员工，并给出智能体的正式定义与两组实证数据；2.2 节回答"像什么"：用"员工接活"的类比拆出智能体工作的六步闭环——接任务、规划、调工具、执行、自查、纠错，并由此划出一条关键界线：给系统接上 AI，与造出一名能独立交付的数字员工，是两码事；2.3 节把镜头从"一个员工"拉到"一个团队"：为什么 2026 年的主流趋势是多智能体协作，主管智能体如何拆解与分派任务，以及"上团队"为什么不等于"出结果"；2.4 节回答"用在哪"：给出一把任务级的尺子——流程可理清、结果可核验两个维度，正例与反例各自落在哪个格子里。

需要事先说明本章的分工边界：本章只回答"是什么、像什么、用在哪"，全部用管理者熟悉的类比和商业语言，不进入技术机制。至于智能体"怎么做到的、会在哪里失灵"——思维链与推理、工具调用、RAG、记忆与上下文工程、MCP 与 A2A 协作协议、安全边界——这些机制拆解统一放在[第五章](../05_agent_tech/README.md)，本章相关处只做链接，不重复展开。同样，"值不值得做"的经济判据在 7.5 节，"如何逐步放权"的落地方法在 9.5 节。读完本章，管理者应当能在会议室里准确地使用"智能体"这个词，并对任何一个候选场景当场做出"能不能交给它"的初步判断。
