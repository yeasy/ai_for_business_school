# 第三章 为什么是现在：拐点的经济学

第一、二章回答了"智能体是什么、能做什么"。本章回答一个对经营者更要紧的问题：为什么拐点偏偏在此刻到来——早两年为什么不成，再等两年又会失去什么。这不是一道技术题，而是一道经济学题：能力、成本与基础设施三条曲线在何时交汇，决定了一项技术从"有意思"变成"算得过账"；而一旦算得过账，行动与观望的风险就不再对称。

全章四节展开。3.1 节给出拐点的构成：模型能力跨过可用门槛、完成同一件事的成本快速下探、工具调用与协议生态等配套设施成形——三个条件第一次在同一时间窗口凑齐，并叠加中国产业与政策面的顺风。3.2 节把成本这笔账算细：不要盯词元单价，要算"完成同等难度一件事"的总成本；同等能力的价格在坍塌、前沿能力的价格在攀升，两条曲线的分化本身就是一道战略选择题。3.3 节面向经历过信息化浪潮的中国企业，比较两轮浪潮的部署逻辑：信息化横向铺开、动辄数年、伤筋动骨，AI 化纵向切入、天级见效、可以回退——门槛结构变了，正确的启动方式也随之改变。3.4 节用柯达与诺基亚两个案例，结合克里斯坦森的颠覆式创新理论，说明组织惯性如何让技术在手的在位者输掉转型，并给出本章最重要的判断：早动手是小额可逆的试错，晚追赶是数量级的代价——风险是不对称的。

分工边界：本章只回答"时机"。支撑拐点的技术机理（推理、工具调用、协议）见第四、五章；成本收益的完整算账框架见第七章；从试点到规模化的落地方法见第九章；防御、延伸、颠覆的战略选择见第十章——本章 3.4 节的颠覆理论与 [10.3](../10_strategy/10.3_three_strategies.md) 的"颠覆"战略互为呼应：这里讲"为什么不能等"，那里讲"主动颠覆怎么做"。
