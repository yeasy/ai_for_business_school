# 第五章 智能体核心技术拆解

第二章把智能体定义为“能交付的数字员工”，回答了“是什么、像什么、用在哪”；本章打开引擎盖，回答另外两个问题：**怎么做到的，以及会在哪失灵**。对管理者而言，懂这两个问题不是为了写代码，而是为了算账与授权——不了解机制，就无法判断供应商的承诺哪些可信；不了解失灵模式，就无法为智能体设定合理的权限与验收标准。

全章六节，按“想—做—查—记—连—防”展开：

- [5.1 思维链与推理模型](5.1_cot.md)：模型如何“先想后答”，以及“算力换质量”背后的成本账；
- [5.2 让 AI 动手干活：工具调用](5.2_tool_use.md)：模型如何从生成文本走向操作系统与软件；
- [5.3 RAG：让 AI 用上企业知识](5.3_rag.md)：企业私有知识如何安全地供给模型；
- [5.4 让 AI 记住企业：记忆与上下文工程](5.4_context_eng.md)：为什么“给对上下文”往往比“换更强模型”更有效；
- [5.5 智能体的“通用插座”：MCP 与 A2A](5.5_mcp_a2a.md)：两个开放协议如何终结集成乱局，及其商业含义；
- [5.6 安全边界：提示注入与越权风险](5.6_security.md)：新攻击面与四道技术防线。

**分工边界**：本章与第二章互为表里——[5.2](5.2_tool_use.md) 深化 [2.2 六步闭环](../02_agent/2.2_work_loop.md)的执行机制，[5.5](5.5_mcp_a2a.md) 深化 [2.3 多智能体协作](../02_agent/2.3_multi_agent.md)的通信底座，彼此链接、不重复展开。[5.6](5.6_security.md) 只讲攻击面与技术防线：如何把防线落成项目管理动作见 [9.5](../09_landing/9.5_trust_control.md)，企业级风险分类与治理制度见 [12.1](../12_governance/12.1_risk_map.md)。每节均以一段“管理含义”收尾，把技术机制翻译成决策语言。

阅读本章不需要工程背景。判断的尺子只有一把：凡是讲不清机制、说不出失灵条件的方案，都还不到签字的时候。
